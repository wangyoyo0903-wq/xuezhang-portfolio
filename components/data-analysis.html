DAU / MAU

日/月活跃用户数

什么是DAU/MAU？

DAU（Daily Active Users）是日活跃用户数，MAU（Monthly Active Users）是月活跃用户数。这是衡量产品用户规模最基础的指标。

DAU/MAU = 用户粘性（理想值 > 20%）

📌 实际案例

某社交App：DAU 100万，MAU 500万

粘性 = 100万/500万 = 20%，说明用户平均每月活跃6天

📋 如何定义"活跃"？

  • 启动活跃：打开App即算活跃（最宽松）
  • 行为活跃：完成核心操作才算（更精准）
  • 时长活跃：使用时长超过阈值（如30秒）

💡 产品经理需要知道的

1. DAU/MAU比值反映用户粘性，越高越好

2. 不同产品类型的DAU/MAU基准不同

3. 要关注趋势变化，而非单一数值

用户规模 粘性指标 核心指标
📈

留存率

用户回访比例

什么是留存率？

留存率衡量用户在一段时间后是否还在使用产品。是评估产品质量和用户忠诚度的核心指标。

留存率 = 回访用户数 / 新增用户数 × 100%

📌 常见留存指标

次日留存：第2天回来的用户比例

7日留存：第7天回来的用户比例

30日留存：第30天回来的用户比例

📋 行业基准参考

产品类型 次日留存 7日留存 30日留存
社交类 40-60% 20-40% 10-20%
工具类 30-50% 15-30% 5-15%
游戏类 30-50% 15-25% 5-10%

💡 产品经理需要知道的

1. 次日留存反映新手引导效果

2. 7日留存反映核心功能吸引力

3. 留存率低需要分析具体流失原因

用户忠诚 产品质量 流失分析
🎯

转化率

目标完成比例

什么是转化率？

转化率是用户完成预期目标的比例。不同产品有不同的转化目标，如注册、购买、分享等。

转化率 = 完成目标用户数 / 总用户数 × 100%

📌 常见转化场景

注册转化：访客 → 注册用户

购买转化：浏览商品 → 完成购买

付费转化：免费用户 → 付费用户

📋 提升转化率的方法

  • 简化流程：减少操作步骤，降低门槛
  • 优化引导：清晰的指引和提示
  • 建立信任：展示评价、保障信息
  • 激励措施：优惠券、限时活动等
  • A/B测试：对比不同方案效果

💡 产品经理需要知道的

1. 转化率是衡量产品效率的核心指标

2. 要关注每个环节的转化，找出瓶颈

3. 小幅提升转化率可能带来巨大收益

漏斗分析 A/B测试 用户行为
💰

ARPU / ARPPU

用户平均收入

什么是ARPU？

ARPU（Average Revenue Per User）是每用户平均收入，ARPPU是每付费用户平均收入。用于衡量产品的变现能力。

ARPU = 总收入 / 总用户数

📌 计算示例

某游戏月收入1000万，月活用户500万

ARPU = 1000万 / 500万 = 2元/用户/月

若付费用户50万，ARPPU = 1000万 / 50万 = 20元

📋 不同产品的ARPU参考

产品类型 ARPU范围 主要变现方式
游戏 5-50元/月 内购、广告
电商 50-500元/月 交易佣金
工具 1-10元/月 会员、广告

💡 产品经理需要知道的

1. ARPU反映产品的商业价值

2. 提升ARPU需要优化付费设计

3. 要平衡用户体验和商业化

变现能力 商业模式 付费设计
💎

LTV

用户生命周期价值

什么是LTV？

LTV（Life Time Value）是用户从开始使用到离开产品期间创造的总价值。帮助判断可以花多少钱获取用户。

LTV = ARPU × 用户生命周期（月）

📌 计算示例

用户平均ARPU为10元/月

用户平均使用时长为12个月

LTV = 10 × 12 = 120元

📋 LTV与CAC的关系

  • CAC：用户获取成本（Customer Acquisition Cost）
  • LTV > CAC：商业模式可持续
  • LTV > 3×CAC：健康的商业模式
  • LTV < CAC：每获取一个用户都在亏损

💡 产品经理需要知道的

1. LTV帮助确定获客预算上限

2. 提升LTV需要提高留存和ARPU

3. 不同用户群体的LTV可能差异很大

用户价值 获客成本 ROI

漏斗分析

转化路径分析

什么是漏斗分析？

漏斗分析用于分析用户在完成目标过程中的转化情况，找出流失严重的环节进行优化。

📌 电商购买漏斗示例

首页浏览 → 商品详情 → 加入购物车 → 结算 → 支付

100% → 40% → 20% → 15% → 12%

📋 漏斗分析的关键点

  • 定义关键路径：明确用户完成目标需要经过的步骤
  • 计算各环节转化率：找出转化率最低的环节
  • 分析流失原因：为什么用户在这一步离开
  • 针对性优化：改进问题环节
  • 持续监测：验证优化效果

💡 产品经理需要知道的

1. 漏斗最窄的环节是优化的重点

2. 不同用户群体的漏斗可能不同

3. 要结合定性分析了解流失原因

用户分群

User Segmentation

什么是用户分群？

将用户按照特定特征分成不同群体，分析不同群体的行为差异，制定针对性的产品策略。

📌 常见分群维度

人口属性：年龄、性别、地域

行为特征：活跃度、付费习惯

使用场景：使用时间、设备类型

📋 用户分群的应用

  • 精准营销：针对不同群体推送不同内容
  • 产品优化：优先满足核心用户群体需求
  • 运营策略：针对流失风险用户进行召回
  • 定价策略：不同群体采用不同定价

💡 产品经理需要知道的

1. 分群要有明确的业务目的

2. 分群维度要可操作、可触达

3. 定期更新用户分群策略

用户画像 精准营销 个性化
⭐

NPS净推荐值

Net Promoter Score

什么是NPS？

NPS通过"你有多大可能向朋友推荐这个产品"（0-10分）来衡量用户忠诚度和口碑。

NPS = 推荐者比例 - 贬损者比例

📌 用户分类

推荐者（9-10分）：忠诚用户，会主动推荐

中立者（7-8分）：满意但不热情

贬损者（0-6分）：不满意，可能负面传播

📋 NPS解读

NPS范围 评价
> 50 优秀，用户非常忠诚
0 ~ 50 良好，有改进空间
< 0 需要重点关注和改进

💡 产品经理需要知道的

1. NPS是衡量用户满意度的简单有效指标

2. 要关注贬损者的具体原因

3. NPS趋势比单次数值更重要

用户满意度 口碑传播 用户忠诚
⏱️

使用时长

Time on Page / Session

什么是使用时长？

用户在产品中停留的时间。反映用户对产品的参与度和粘性，但需要结合场景解读。

📌 不同场景的解读

内容类产品：时长越长越好，说明内容有吸引力

工具类产品：时长短可能更好，说明效率高

电商类产品：适中为好，过长可能找不到商品

📋 相关指标

  • 单次使用时长：每次打开App的平均使用时间
  • 日均使用时长：每天平均使用时间
  • 页面停留时长：在特定页面停留的时间
  • 会话次数：每天打开App的次数

💡 产品经理需要知道的

1. 时长指标要结合产品类型解读

2. 关注时长分布，而非只看平均值

3. 时长异常变化可能是产品问题的信号

用户粘性 参与度 内容质量
🔗

分享率

Share Rate

什么是分享率？

用户主动分享产品内容的比例。反映产品的社交传播能力和用户认可度。

分享率 = 分享用户数 / 活跃用户数 × 100%

📌 提升分享率的方法

内容激励：创造值得分享的优质内容

功能便捷：一键分享到各平台

利益驱动：分享获得奖励

📋 分享相关的指标

  • 分享率：有多少用户进行了分享
  • 分享次数：平均每人分享几次
  • 分享回流：通过分享带来的新用户
  • K因子：每个用户平均带来多少新用户

💡 产品经理需要知道的

1. 分享是低成本获取用户的方式

2. 要设计合适的分享触发场景

3. 分享内容要有吸引力

病毒传播 社交裂变 用户增长
⚠️

崩溃率

Crash Rate

什么是崩溃率？

App在使用过程中崩溃的比例。是衡量产品质量和稳定性的关键指标，直接影响用户体验。

崩溃率 = 崩溃次数 / 启动次数 × 100%

📌 行业标准

优秀：崩溃率 < 0.2%

良好：崩溃率 < 0.5%

需改进：崩溃率 > 1%

📋 崩溃分析要点

  • 崩溃类型：什么原因导致的崩溃
  • 崩溃场景：在什么操作时崩溃
  • 设备分布：哪些设备更容易崩溃
  • 版本分布：哪个版本崩溃率更高

💡 产品经理需要知道的

1. 崩溃率是产品质量的红线指标

2. 新版本发布后要重点监控崩溃率

3. 崩溃会直接导致用户流失和差评

产品质量 用户体验 稳定性

📚 数据分析学习建议

产品经理的数据分析能力进阶路径：

  1. 第一阶段：理解核心指标的含义和计算方法
  2. 第二阶段：学会看数据报表，发现问题
  3. 第三阶段：能够提出数据需求，设计埋点方案
  4. 第四阶段：用数据驱动产品决策，验证效果
💡 记住：数据只是工具，不是目的。产品经理要学会从数据中发现问题、验证假设，但最终还是要回归用户价值和产品本质。
